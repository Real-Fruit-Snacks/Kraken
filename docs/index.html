01 Premise

An OPSEC-first C2 for operators who assume every byte gets logged.

Kraken is a Rust-built command-and-control framework. The implant is small, encrypted by default, and capability-loaded at runtime so what lands on disk reveals very little about what the operator actually intends to do.

This is not yet operational — Kraken is under active development. APIs, protocols, and module surfaces will change. Read the architecture decisions in docs/DECISIONS.md before building anything against it.

02 Specs

What's in the box.

CRYPTO
X25519 ECDH key exchange, HKDF-SHA256 derivation, AES-256-GCM with per-message nonces, Ed25519 signing.
XPORTS
HTTPS · TCP · SMB · DNS with automatic fallback chains. Selection at compile-time or runtime.
CORE
Minimal hardened implant ~50 KB. All capabilities loaded as runtime modules.
MESH
Peer discovery and relay routing. Implants communicate through each other when direct C2 is blocked.
BOF
Rust COFF loader. Run Cobalt Strike Beacon Object Files unmodified.
STACK
9-crate Cargo workspace · gRPC (tonic) · SQLite (sqlx) · operator CLI + React Web UI.
03 Quickstart

Build, server, operator.

Kraken's build is driven by just. The full path: setup  →  protobuf gen  →  build  →  server  →  operator.

# Prereqs: Rust 1.81+, protoc 3.20+, SQLite dev libs
$ git clone https://github.com/Real-Fruit-Snacks/Kraken.git
$ cd Kraken
$ just setup && just proto && just build

# Start the team server (gRPC :50051 · HTTPS :443)
$ just server

# Operator CLI in another terminal
$ just operator -- --server 127.0.0.1:50051

# Test without real implants
$ just sim -- --server https://127.0.0.1:443 --interval 5

05 Architecture

Three tiers, nine crates.

Operator (CLI + gRPC client) connects to Teamserver (gRPC + HTTP listener + SQLite) which manages Implants (minimal core + modules). All communication encrypted end-to-end.

crates/
common/        // shared traits, ID types
crypto/        // X25519 · GCM · HKDF · Ed25519
protocol/      // protobuf · tonic · prost
config/        // compile-time config baking
db/            // SQLite via sqlx
server/        // teamserver: gRPC + HTTP + audit
operator/      // CLI: commands · sessions
implant-core/  // minimal kernel: dispatch · transport
implant-sim/   // simulator for testing
06 Authorization

Without permission, don't.

Kraken is built for engagements that are scoped, written, and signed. The README's first paragraph is the rules of engagement, not the install instructions.

Vulnerabilities go through private security advisories, never public issues. The framework includes no ransomware, wiper, or supply-chain capability and never will.

All implant configuration bakes at compile time — no runtime args, no environment lookups, no secrets on disk.
