FEATURES

OPSEC-first by design

X25519 key exchange. AES-256-GCM encryption. Modular implant with 21 capability modules. Mesh networking for segmented environments.

Encrypted Communications

X25519 ECDH key exchange with HKDF-SHA256 derivation and AES-256-GCM bulk encryption. Server public key baked at compile time prevents MITM. Per-message nonces with server-side replay protection.

// Key exchange flow
Implant → ephemeral X25519 → shared secret
→ HKDF-SHA256 → AES-256-GCM session keys
→ per-message nonce → encrypted payload
21
Implant Modules

Multi-Transport

HTTP/HTTPS, TCP, SMB, and DNS with automatic fallback chains. Implants cycle transports on failure with configurable intervals.

Modular Implant

Minimal hardened core with runtime-loadable signed modules. Shell, file ops, BOF, mesh, SOCKS5, injection, evasion, tokens, credentials, lateral movement, and more.

Mesh Networking

Peer discovery, relay, and topology computation. Compromised hosts act as relay nodes for network segments without direct C2 connectivity.

ARCHITECTURE

Nine-crate workspace

Each crate owns a single responsibility with explicit pub API boundaries. Clean separation between crypto, transport, protocol, and UI.

1

Crypto & Protocol

X25519 ECDH, AES-256-GCM, HKDF-SHA256, Ed25519 in the crypto crate. Protobuf message types generated by tonic/prost in protocol. Shared traits and error types in common.

2

Teamserver

gRPC services (tonic) for operator communication. HTTP/HTTPS listeners (axum) for implant registration and check-in. SQLite database via sqlx. Append-only audit logging. RBAC and session management.

3

Implant Core

Minimal kernel handling registration, check-in loop, task dispatch, and transport chains. All configuration baked at compile time. Module registry loads capabilities on demand. Ed25519 signature verification for modules.

4

Operator CLI

Command-line interface with structured output. gRPC client for teamserver communication. Task dispatch, result rendering, and session management.

GETTING STARTED

Build from source

Requires Rust 1.75+, protoc 3.20+, and SQLite development libraries. Build the entire workspace or individual components.

build.sh
$ git clone https://github.com/Real-Fruit-Snacks/Kraken.git
$ cd Kraken

# Build all components
$ just build

# Or build individually
$ just server
$ just operator
$ just implant-linux-x64
$ just implant-windows-x64
usage.sh
# Terminal 1 — Teamserver
$ just server

# Terminal 2 — Operator CLI
$ just operator

# Terminal 3 — Implant simulator
$ just sim

DETECTION AWARENESS

Know the boundaries

Evades

  • Passive traffic analysis — X25519 + AES-256-GCM encryption
  • Static signature scanning — compile-time config, no strings
  • DPI inspection — multi-transport with protocol mimicry
  • Timing analysis — jittered check-in with sleep masking
  • Module detection — Ed25519 signed, loaded on demand

Visible To

  • strace, eBPF — syscall tracing
  • EDR / XDR — behavioral analysis and heuristics
  • Network monitoring — connection metadata and flow
  • Memory forensics — RAM inspection and dump analysis
  • SELinux, AppArmor — mandatory access control
