FEATURES

OPSEC-first by design

Encrypted communications, modular implants, mesh networking, and a professional operator interface.

Encrypted Communications

X25519 ECDH key exchange with HKDF-SHA256 derivation. AES-256-GCM bulk encryption with per-message nonces. Nonce replay protection on the server.

# Keys baked at compile time
$ just implant-linux-x64
X25519 keypair embedded — Ctrl+C to disconnect
9
Rust Crates

Modular Implant

Minimal hardened core with runtime-loadable signed modules. Shell, file ops, SOCKS proxy, mesh relay, BOF execution.

Mesh Networking

Peer discovery, relay routing, and topology computation. Implants communicate through each other when direct C2 is blocked.

Relay & SOCKS5

Port forwarding, multi-hop chain relay, and proxychains-compatible SOCKS5 proxy server.

ARCHITECTURE

Nine-crate workspace

Each crate owns a single responsibility. Common traits shared, crypto isolated, protocol generated from protobuf.

1

CLI & Config

clap-based argument parsing with nc-compatible shortcuts. Global flags (--tls, --encrypt, --obfuscate) apply to any mode. TOML config at ~/.config/flux/.

2

Teamserver

gRPC services via tonic, HTTP/HTTPS listener via axum, SQLite state, audit logging. Multi-operator RBAC and session management.

3

Implant Core

Minimal kernel: registration, check-in loop, task dispatch, transport chain. Runtime-loadable modules implement Module trait.

4

Operator TUI

ratatui terminal UI with Catppuccin theming. Real-time event streaming, keybinding navigation, structured result rendering.

GETTING STARTED

Build and deploy

Nine-crate Cargo workspace. Build everything with a single command.

build
$ git clone https://github.com/Real-Fruit-Snacks/Kraken.git
$ cd Flux && cargo build --release

# Static binary for target deployment
$ cross build --target x86_64-unknown-linux-musl --release

$ ./target/release/flux --version
flux 0.1.1
operate
# Start teamserver
$ just server

# Connect operator
$ just operator -- --server 127.0.0.1:50051

# Test with simulator
$ just sim -- --server https://localhost:443

kraken> shell whoami
DETECTION AWARENESS

Know the boundaries

Hidden From

  • Compile-time config — no runtime key exchange visible
  • Encrypted check-in — blends with normal HTTPS
  • Typed results — structured data, not raw shell output
  • Module signing — Ed25519 prevents tampering
  • Jittered check-in — randomized intervals

Visible To

  • EDR/XDR — behavioral analysis
  • Network flow analysis — connection metadata
  • Memory forensics — implant resident in RAM
  • JA3/JA4 — TLS client fingerprinting
  • Kernel monitoring — ETW, eBPF, Sysmon
